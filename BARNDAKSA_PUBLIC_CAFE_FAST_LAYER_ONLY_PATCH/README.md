# BARNDAKSA_PUBLIC_CAFE_FAST_LAYER_ONLY_PATCH

باتش يضيف طبقة سرعة داخلية للفرع الإلكتروني فقط `/c/[slug]` بدون تغيير منطق الحفظ أو الدفع أو الحجوزات أو الولاء.

## طريقة التشغيل
من جذر المشروع:

```bash
node BARNDAKSA_PUBLIC_CAFE_FAST_LAYER_ONLY_PATCH/apply-patch.cjs
npm exec tsc -- --noEmit
npm run build
```

## ما الذي يفعله؟
- يضيف API سريع موحد: `/api/public/cafe/[slug]/fast`.
- يجمع بيانات الفرع والثيم والشعار والميزات والمنتجات والعروض والفروع والحجوزات المتاحة في طلب واحد.
- يضيف كاش داخلي في السيرفر لتقليل ضربات Supabase على الصفحات العامة.
- يضيف كاش في المتصفح/localStorage للفرع الإلكتروني فقط حتى تظهر الصفحة فورًا ثم تتحدث بالخلفية.
- يترك Supabase مصدر الحقيقة ولا يغيّر أي عمليات حساسة.
- لا يحتاج SQL.

## ملاحظات
إذا تغيّر منتج أو شعار أو حجز من لوحة التحكم، الزائر قد يرى النسخة المخزنة لحظيًا ثم تتحدث بالخلفية تلقائيًا. الطبقة مصممة كـ fallback: إذا فشل الكاش أو الـ API السريع، يرجع النظام للمسارات القديمة بدل ما تتعطل الصفحة.
