# Dynamic Features And Packages

## الهدف

يعتمد الديمو الآن على سجل مركزي للميزات بدل تعريف عناصر لوحة العلامة والباقات في أماكن متعددة. أي خدمة تظهر في شريط لوحة العلامة يجب أن تكون مسجلة في سجل الميزات، ومنه تقرأ الباقات وصفحات الإدارة.

## المصدر الواحد

السجل موجود في:

```text
lib/platform/feature-registry.ts
```

ويصف كل ميزة بهذه البيانات: المعرف، الاسم العربي، الاسم الإنجليزي، الوصف، التصنيف، المسار، الظهور في الشريط الجانبي، قابلية الإسناد للباقات، التفعيل الافتراضي، مستوى الباقة المقترح، مفتاح الأيقونة، مستوى الخطورة، الحالة، الترتيب، والاعتماديات.

المساعدات النقية موجودة في:

```text
lib/platform/feature-access.ts
```

وتوفر:

```text
getAllPlatformFeatures()
getPackageAssignableFeatures()
getSidebarFeaturesForBrand(context)
getPlanIncludedFeatures(planId, plans)
getBrandFeatureOverrides(brandId)
getEffectiveBrandFeatureAccess(planFeatures, brandOverrides)
```

## قراءة الشريط الجانبي

شريط لوحة العلامة يقرأ الميزات من السجل عبر مساعدات الوصول. العناصر الأساسية مثل لوحة التحكم والإعدادات والاشتراك مفعلة افتراضيًا، بينما الخدمات التشغيلية تظهر حسب الباقة. الكاشير وبرندة المالية يبقيان ظاهرين كخدمات تشغيلية/معاينة مع حالة قفل عند عدم وجودهما في الباقة.

## قراءة الباقات

صفحة إدارة الباقات تستخدم الميزات القابلة للإسناد فقط من السجل. تعرض كل ميزة مع تصنيفها، مسارها، حالتها، وهل هي قابلة للإسناد، ثم تسمح بتضمينها أو إزالتها من الباقة عبر الحفظ الحالي الموجود في المنصة.

## تجاوزات العلامة

المطلوب مهنيًا هو أن يستطيع الأدمن تعطيل أو تفعيل خدمة لعلامة محددة دون تغيير الباقة. مثال: علامة على الباقة الأساسية لكن الأدمن يعطل "المنيو" لهذه العلامة فقط.

الواجهة تعرض الآن قسم:

```text
صلاحيات وخدمات العلامة
```

وفيه:

- الباقة الحالية للعلامة.
- الخدمات المشمولة في الباقة.
- الخدمات المقفلة يدويًا.
- الخدمات المفعلة يدويًا.
- النتيجة النهائية التي تظهر لصاحب العلامة.

## ما هو مطبق الآن

- سجل مركزي للميزات.
- ربط الشريط الجانبي بالسجل.
- ربط إدارة الباقات بالسجل.
- جعل "برندة المالية" خدمة قابلة للإسناد للباقات بحالة معاينة وخطورة مالية.
- عرض حالة الصلاحيات على مستوى العلامة في الإدارة.
- حفظ الباقات الحقيقي ما زال يستخدم الدوال الحالية.

## ما يحتاج دعم قاعدة بيانات

لا يوجد في الكود الحالي دعم مثبت لتجاوزات الميزات لكل علامة. لذلك عناصر التحكم في صفحة العلامة معروضة كمعاينة ومقفلة بدل حفظ وهمي.

الدعم المقترح، للتوثيق فقط ودون تنفيذ SQL هنا:

```text
brand_feature_overrides
- id uuid primary key
- cafe_id uuid not null references cafes(id)
- feature_id text not null
- enabled boolean not null
- reason text null
- created_by uuid null references profiles(id)
- created_at timestamptz not null default now()
- updated_at timestamptz not null default now()
- unique(cafe_id, feature_id)
```

أو إذا كان المشروع يفضل تخزينًا مدمجًا:

```text
cafe_feature_overrides
- cafe_id uuid
- feature_id text
- override text check in ('enabled', 'disabled')
- updated_by uuid
- updated_at timestamptz
```

بعد إضافة الدعم، يجب تحديث:

```text
lib/data/feature-entitlements.ts
lib/data/admin.ts
app/actions/admin.ts
```

ليتم دمج ميزات الباقة مع تجاوزات العلامة قبل إرجاع الصلاحيات العامة أو صلاحيات صاحب العلامة.
