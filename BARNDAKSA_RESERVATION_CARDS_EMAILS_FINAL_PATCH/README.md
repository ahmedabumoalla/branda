# BARNDAKSA_RESERVATION_CARDS_EMAILS_FINAL_PATCH

يشمل هذا الباتش:

- عرض بطاقات الحجز في صفحة عميل العلامة من نفس خدمات الحجز التي تديرها العلامة في لوحة التحكم.
- إظهار بطاقات الحجز قبل تسجيل الدخول، ثم يطلب من العميل تسجيل الدخول لإكمال الطلب.
- إرسال إيميل للعميل عند إرسال طلب الحجز.
- إرسال إيميل للعلامة التجارية عند وصول حجز جديد.
- إرسال إيميل للعميل عند قبول/رفض/طلب تعديل الحجز مع تفاصيل الحجز والسبب إن وجد.

## التشغيل

من جذر المشروع:

```bash
node BARNDAKSA_RESERVATION_CARDS_EMAILS_FINAL_PATCH/apply-patch.cjs
npm exec tsc -- --noEmit
npm run build
```

ملاحظة: إرسال الإيميلات يتطلب ضبط `RESEND_API_KEY` و `RESEND_FROM_EMAIL`، وبريد العلامة يؤخذ من إعدادات الكوفي `owner_email`.
